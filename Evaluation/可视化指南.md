## 快速开始

### 单文件快速生成

```bash
python Evaluation\plot_results.py --input results/evaluations/clearrag/clear100_001.json
```

### 多框架对比

```bash
python Evaluation\plot_results.py \
    --input \
        results/evaluations/clearrag/clear100_001.json \
        results/evaluations/lightrag/lightrag_all_001.json \
    --output figures/ \
    --type all \
    --prefix retrieval_comparison
```

## 命令行参数

| 参数 | 简写 | 类型 | 必需 | 默认值 | 说明 |
|------|------|------|------|--------|------|
| `--input` | `-i` | 多值 | 是 | - | 输入JSON文件路径（支持多个文件） |
| `--output` | `-o` | 单值 | 否 | `figures/` | 输出目录 |
| `--type` | `-t` | 单值 | 否 | `bar` | 图表类型 |
| `--title` | | 单值 | 否 | 无 | 图表标题 |
| `--prefix` | | 单值 | 否 | `evaluation` | 输出文件名前缀 |

### 图表类型选项

| 值 | 说明 |
|----|------|
| `bar` | 分组柱状图 |
| `radar` | 雷达图（每个指标生成一张） |
| `heatmap` | 热力图 |
| `line` | 折线对比图 |
| `all` | 生成全部图表类型 |

## 图表类型说明

### 1. 分组柱状图 (bar)

按问题类型分组，展示不同框架在各指标上的得分对比。

**适用场景：**
- 直观比较不同框架的性能差异
- 论文中的主要对比图
- 强调具体数值差异

**输出：** `{prefix}_bar.png`

### 2. 雷达图 (radar)

多维度综合对比，展示各框架的整体性能轮廓。

**适用场景：**
- 展示框架的综合能力
- 多维度性能平衡分析
- 论文中的概览图

**输出：** `{prefix}_radar_{metric}.png`（每个指标一张）

### 3. 热力图 (heatmap)

矩阵形式展示框架×问题类型的综合得分。

**适用场景：**
- 快速定位最优/最差组合
- 大规模对比（多框架×多类型）
- 补充材料

**输出：** `{prefix}_heatmap.png`

### 4. 折线图 (line)

展示各框架在不同问题类型上的性能变化趋势。

**适用场景：**
- 分析难度递增时的性能变化
- 趋势分析
- 辅助说明图

**输出：** `{prefix}_line.png`

## 使用示例

### 示例1：检索评估对比

```bash
# ClearRAG vs LightRAG vs LinearRAG 检索性能对比
python Evaluation\plot_results.py `
    --input `
        results/evaluations/clearrag/clear100_001.json `
        results/evaluations/lightrag/lightrag_all_001.json `
        results/evaluations/linearrag/retrieval/retrieval_Medical_完整数据集.json `
    --output figures/retrieval/ `
    --type all `
    --prefix retrieval_medical_100 `
    --title "Retrieval Performance on Medical-100"
```

### 示例2：生成评估对比

```bash
# 生成质量对比
python Evaluation\plot_results.py \
    --input \
        results/evaluations/clearrag/clear_generation_medical_100_001_2026-3-3.json \
        results/evaluations/lightrag/light_generation__Medical_100_2026-3-3.json \
    --output figures/generation/ \
    --type all \
    --prefix generation_medical_100 \
    --title "Generation Performance on Medical-100"
```

### 示例3：仅生成柱状图

```bash
python Evaluation\plot_results.py \
    --input results/evaluations/clearrag/clear100_001.json \
    --output figures/ \
    --type bar \
    --prefix clearrag_bar
```

### 示例4：Windows PowerShell 格式

```powershell
python Evaluation\plot_results.py `
    --input `
        results/evaluations/clearrag/clear100_001.json `
        results/evaluations/lightrag/lightrag_all_001.json `
    --output figures/ `
    --type all `
    --prefix retrieval_comp
```

## 输出文件说明

执行后会在输出目录生成以下文件：

```
{output_dir}/
├── {prefix}_bar.png                    # 分组柱状图
├── {prefix}_radar_context-relevancy.png # Context Relevancy 雷达图
├── {prefix}_radar_evidence-recall.png   # Evidence Recall 雷达图
├── {prefix}_heatmap.png                 # 热力图
├── {prefix}_line.png                    # 折线对比图
└── {prefix}_summary.md                  # Markdown 汇总表格
```

## 支持的评估类型

### 检索评估指标

| 指标 | 显示名称 | 说明 |
|------|----------|------|
| `context_relevancy` | Context Relevancy | 上下文相关性 |
| `evidence_recall` | Evidence Recall | 证据召回率 |

### 生成评估指标

| 指标 | 显示名称 | 适用问题类型 |
|------|----------|--------------|
| `rouge_score` | ROUGE-L | Fact Retrieval, Complex Reasoning |
| `answer_correctness` | Answer Correctness | 全部类型 |
| `coverage_score` | Coverage | Contextual Summarize, Creative Generation |
| `faithfulness` | Faithfulness | Creative Generation |

### 问题类型

| 类型 | 简写 | 说明 |
|------|------|------|
| Fact Retrieval | FR | 事实检索 |
| Complex Reasoning | CR | 复杂推理 |
| Contextual Summarize | CS | 上下文摘要 |
| Creative Generation | CG | 创意生成 |

## 常见问题

### Q1: 中文显示乱码

确保系统安装了 SimHei 字体，或修改脚本中的字体配置：

```python
plt.rcParams['font.sans-serif'] = ['SimHei', 'DejaVu Sans']
```

### Q2: 找不到模块

确保已安装依赖：

```bash
pip install matplotlib numpy
```

### Q3: 如何添加新框架

在脚本的 `COLORS` 字典中添加新框架的颜色配置：

```python
COLORS = {
    'newframework': '#FF5722',  # 添加新颜色
    ...
}
```

### Q4: 输出目录不存在

脚本会自动创建输出目录，无需手动创建。
