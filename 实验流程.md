## 2. 配置实验

直接复制已有配置文件 `configs/experiment.yaml`：到runs目录下,命名方式:{framework}_{dataset}_{sample}.yaml

```yaml
run:
  framework: lightrag          # 框架名称

dataset:
  subset: hotpotqa_100         # 数据集子集
  sample: 10                   # 问题采样数

embedding:
  type: local
  model: BAAI/bge-large-en-v1.5
```


**配置项值得注意的地方**：
- 数据集问题采样数：`sample`:第一次索引需要使用一个比较小的问题采样数，如 10,避免索引出现问题导致后续的查询预测不准确,这里可以修改的前要条件是: 索引成功后没有报错，再修改为较大的问题采样数,比如NULL
- 嵌入模型:优先使用本地模型，如 BAAI/bge-large-en-v1.5,中间遇到兼容性问题再去切换api方式的embedding-3
- 关于clearrag的neo4j设置:只需要修改的是:database,这里的名称有一个规范,就是取用一个和当前数据集相关的名称,
例子:使用的是muisque_100,数据库名称就是:muisque-100-001,001表示的是第一次运行该数据集,以此类比,并且需要手动去neo4j(localhost:7474)中创建一个数据库,名称为:muisque-100-001,然后在configs/experiment.yaml中修改database的值为:muisque-100-001
- skip_build:先检查一遍对应的目录是否有已经构建好的索引,第一次索引需要设置为false,后续相同的数据集可以设置为true
- clearrag框架的参数可以修改:agnent的最大调用工具次数max_tool_calls(3~10),以及段落检索的模式passage_retrieval_mode:# vector / pagerank / none,通常是pagerank和none

## 3. 运行实验
0. 脚本Examples/run_from_yaml.py支持的CLI参数:
```bash
python Examples/run_from_yaml.py --help
```
建议运行方式:复制yaml文件复制到runs目录下,并修改对应的参数,然后运行再去通过--config 参数运行脚本

1. 先运行以下命令检查配置是否正确：
```bash
python Examples/run_from_yaml.py --framework clearrag --dry-run
```
2. 正式运行:
```bash
python Examples/run_from_yaml.py --framework clearrag
```
3. 由于先是设置了sample,所以这里需要根据上面的运行结果进行分析,如果上面没有报错,并且查询样本没有出现明显的问题和报错,就需要把sample再修改为NULL,同时需要修改skip_build这个参数为true

4. 运行预测
```bash
python Examples/run_from_yaml.py --framework clearrag
```

**运行流程**：
1. 加载配置 → 解析框架和数据集
2. 获取数据集路径（corpus + questions）
3. 初始化框架适配器
4. **索引构建**：切分文本 → 提取实体/关系 → 构建图谱/向量索引
5. **查询回答**：对每个问题检索相关上下文 → LLM 生成答案
6. 保存结果到 `results/{framework}/{subset}/{run_id}/`

## 4. 评估结果

### 4.1 统一评估（推荐）

**先进行环境变量的设置**:
```bash
$env:LLM_BASE_URL="https://api.deepseek.com"
$env:LLM_API_KEY="可以从configs\experiment.yaml里面找"
$env:LLM_MODEL="deepseek-chat"

```bash
python -m Evaluation.unified_eval --data_file ./results/{run}/predictions.json --output_file ./results/{run}/评估模式.json --report
```

**参数说明**：
- `--data_file`: 预测结果文件（必需）
- `--output_file`: 评估结果输出路径（必需）
- `--detailed`: 输出详细结果（包含每个样本）
- `--report`: 打印格式化报告

**输出指标**：
- 答案指标: `answer_em`, `answer_f1`, `rouge_score`
- 检索质量: `evidence_coverage`
- 支持事实: `sf_em`, `sf_f1`, `joint_em`, `joint_f1`
- 三元组/推理步骤（如适用）

### 4.2 检索评估
```bash
python -m Evaluation.retrieval_eval --mode API --data_file ./results/{run}/predictions.json --output_file ./results/retrieval_eval.json
```

**主要参数**：
- `--mode`: API必需）
- `--data_file`: 预测结果文件（必需）
- `--output_file`: 输出文件路径(要和预测结果文件一致)
- `--model`: LLM模型 默认deepseek-chat
- `--base_url`: API地址
- `--embedding_model`: 嵌入模型（默认 BAAI/bge-large-en-v1.5）
- `--num_samples`: 采样数量（可选）
- `--detailed_output`: 详细输出

### 4.3 生成评估
```bash
python -m Evaluation.generation_eval --mode API --data_file ./results/{run}/predictions.json --output_file ./results/generation_eval.json
```

**参数同检索评估**

### 4.4 索引评估 (比较少,可以忽略)
```bash
python -m Evaluation.indexing_eval --framework lightrag --base_path ./workspace --output ./results/indexing_metrics.txt
```

**参数说明**：
- `--framework`: 框架名称（必需），可选: `microsoft_graphrag`, `lightrag`, `fast_graphrag`, `hipporag2`, `graphml`
- `--base_path`: 索引文件根目录（必需）
- `--folder_name`: 子目录名（hipporag2 必需）
- `--output`: 输出文件路径（可选，默认打印到终端）

评估预测文件和评估指标文件需要在同一个目录下
