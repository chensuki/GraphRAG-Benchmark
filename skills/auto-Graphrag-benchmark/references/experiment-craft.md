# 实验诊断流程（Experiment Craft）

当实验失败或结果不如预期时，使用以下 5 步诊断流程。

---

## 1. 收集失败案例

- 收集具体的失败或糟糕结果示例
- 查看实际输出，不只是聚合指标
- 记录错误消息、异常栈、日志片段

**输出**：失败案例清单

---

## 2. 找到工作版本

- 简化任务、减少复杂度、切换数据
- 从工作版本开始，逐因素添加
- 找到那个单一因素导致失败

**策略**：
- 减少 sample 数量
- 切换到更简单的数据集
- 禁用可选功能

---

## 3. 桥接差距

- 从工作版本开始，渐进添加复杂性
- 找到单一因素，确认根本原因
- 每次只添加一个变量

**验证**：
- A 版本工作 → B 版本失败
- 差异是什么？逐个恢复

---

## 4. 假设与验证

- 列出可能的解释并按可能性排序
- 设计针对性实验验证或消除每个假设
- 通过实验实际确认原因

**假设模板**：
```
假设: {root_cause}
验证方法: {experiment_design}
预期结果: {expected_outcome}
实际结果: {actual_outcome}
结论: {verified_or_rejected}
```

---

## 5. 提出与实施

- 搜索解决该具体原因的技术
- 设计针对性修复并验证
- 记录修复效果

**修复清单**：
- 代码修改
- 配置调整
- 依赖升级
- 数据处理优化

---

## 常见问题模式

| 症状 | 可能原因 | 诊断方向 |
|------|----------|----------|
| 答案为空 | LLM 调用失败 | 检查 API key、base_url |
| 证据覆盖率低 | 检索失败 | 检查 embedding、索引 |
| 格式警告 | 提取失败 | 检查 chunk size、prompt |
| 超时 | 网络/性能问题 | 检查并发数、timeout |
| 内存溢出 | 数据过大 | 减少 corpus_sample |
