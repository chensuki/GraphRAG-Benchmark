## 返回数据类型要求

### 1. 查询结果（BenchmarkQueryResult）

推荐使用dataclass定义标准返回类型：

```python
from dataclasses import dataclass, field
from typing import List, Dict, Any

@dataclass
class BenchmarkQueryResult:
    """标准查询结果类型"""

    # 必需字段
    answer: str                                      # 字符串，必需
    context: List[Dict[str, Any]] = field(default_factory=list)  # 统一格式的上下文列表

    def get_context_list(self) -> List[str]:
        """获取纯文本 context 列表（用于评估脚本）"""
        if not self.context:
            return []
        result = []
        for item in self.context:
            if isinstance(item, dict):
                result.append(item.get("content", ""))
        return [r for r in result if r]

    def get_chunks(self) -> List[Dict[str, Any]]:
        """获取所有 chunk 类型的 context"""
        return [item for item in self.context if isinstance(item, dict) and item.get("type") == "chunk"]

    def get_entities(self) -> List[Dict[str, Any]]:
        """获取所有 entity 类型的 context"""
        return [item for item in self.context if isinstance(item, dict) and item.get("type") == "entity"]

    def get_relationships(self) -> List[Dict[str, Any]]:
        """获取所有 relationship 类型的 context"""
        return [item for item in self.context if isinstance(item, dict) and item.get("type") == "relationship"]
```

**类型要求详细说明**：

#### answer字段
```python
# ✅ 正确
answer: str = "这是答案文本"

# ❌ 错误
answer: List[str] = ["答案"]      # 不能是列表
answer: Dict = {"text": "答案"}   # 不能是字典
answer: None                      # 不能是None（可以用空字符串）
```

**必须**：字符串类型，即使是错误消息也要是字符串

#### context字段

```python
# ✅ 正确：统一格式（字典列表）
context: List[Dict[str, Any]] = [
    {"type": "chunk", "content": "文档内容..."},
    {"type": "entity", "content": "实体名 (类型): 描述", "entity_name": "...", "entity_type": "...", "description": "..."},
    {"type": "relationship", "content": "起点->终点: 描述", "src_id": "...", "tgt_id": "...", "description": "..."}
]
context: List[Dict[str, Any]] = []  # 空列表表示无上下文

# ❌ 错误
context: str = "单个片段"             # 不能是字符串
context: None                        # 不能是None（用空列表）
```

**必须**：字典列表，每个字典包含 `type` 和 `content` 字段

**格式说明**：

| type | content 格式 | 说明 |
|------|--------------|------|
| chunk | `"文档内容..."` | 原始文档片段 |
| entity | `"实体名 (类型): 描述"` | 结构化实体信息 |
| relationship | `"起点->终点: 描述"` | 结构化关系信息 |

**工具函数**（推荐使用）：

```python
from adapters.base import format_entity_content, format_relationship_content

# 格式化实体 content
content = format_entity_content("AlphaGo", "artifact", "AI系统描述...")
# 结果: "AlphaGo (artifact): AI系统描述..."

# 格式化关系 content
content = format_relationship_content("用户", "订单", "创建了")
# 结果: "用户->订单: 创建了"
```

### 2. 字典格式的返回（替代方案）

如果不使用 BenchmarkQueryResult 对象，可以返回字典：

```python
def query(self, question: str, **kwargs) -> Dict[str, Any]:
    """返回字典格式"""
    return {
        "answer": str,                        # 必需，字符串
        "context": List[Dict[str, Any]],      # 必需，统一格式的字典列表
    }
```

**关键要求**：
- `answer` 必须是字符串
- `context` 必须是统一格式的字典列表（包含 `type` 和 `content` 字段）

### 3. 仅返回答案（简化版本）

某些框架可能只返回答案字符串：

```python
def query(self, question: str, **kwargs) -> str:
    """仅返回答案字符串"""
    return "这是答案"
```

**注意**：这种方式需要额外的方法获取上下文，并构建统一格式：

```python
def get_context(self, question: str, **kwargs) -> List[Dict[str, Any]]:
    """获取统一格式的上下文列表"""
    return [
        {"type": "chunk", "content": "片段1"},
        {"type": "chunk", "content": "片段2"}
    ]
```

## 类型验证和转换

### 1. 参数验证函数

**推荐使用灵活的类型转换**（参考评估脚本的实际处理方式）：

```python
from typing import Optional, Any, List

def ensure_context_list(context: Any) -> List[str]:
    """
    灵活的context转换（与评估脚本一致）
    
    参考: Evaluation/retrieval_eval.py:270-277
    """
    if context is None:
        return []
    
    if isinstance(context, str):
        # 字符串转单元素列表
        return [context] if context else []
    
    if isinstance(context, list):
        # 确保列表中的每个元素都是字符串
        return [str(item) if not isinstance(item, str) else item for item in context]
    
    # 其他类型转换为字符串列表
    return [str(context)]

def ensure_string(value: Any, default: str = "") -> str:
    """确保返回字符串"""
    if value is None:
        return default
    if isinstance(value, str):
        return value
    return str(value)

def ensure_int(value: Any, default: int = 10, min_val: int = 1, max_val: int = 100) -> int:
    """确保返回有效整数"""
    if value is None:
        return default
    try:
        int_val = int(value)
        return max(min_val, min(max_val, int_val))
    except (TypeError, ValueError):
        return default
```

### 2. 在框架中使用验证

```python
from adapters.base import (
    BenchmarkQueryResult,
    format_entity_content,
    format_relationship_content
)

class MyRAGFramework:
    def __init__(
        self,
        working_dir: str,
        llm_model_name: str,
        embedding_model_name: str,
        **kwargs
    ):
        # 灵活处理参数
        self.working_dir = ensure_string(working_dir, default="./workspace")
        self.llm_model_name = ensure_string(llm_model_name)
        self.embedding_model_name = ensure_string(embedding_model_name)
        self.top_k = ensure_int(kwargs.get('top_k'), default=10)

    async def aquery(self, question: str, top_k: Optional[int] = None, **kwargs) -> BenchmarkQueryResult:
        """查询方法，返回统一格式"""
        # 处理输入参数
        question = ensure_string(question)
        top_k = ensure_int(top_k, default=self.top_k)

        # 执行查询...
        answer, chunks, entities, relationships = await self._internal_query(question, top_k)

        # 构建统一格式的 context
        context = []

        # 添加 chunks
        for chunk in chunks:
            context.append({"type": "chunk", "content": chunk})

        # 添加 entities
        for entity in entities:
            content = format_entity_content(
                entity.get("name", ""),
                entity.get("type", ""),
                entity.get("description", "")
            )
            if content:
                context.append({
                    "type": "entity",
                    "content": content,
                    "entity_name": entity.get("name", ""),
                    "entity_type": entity.get("type", ""),
                    "description": entity.get("description", "")
                })

        # 添加 relationships
        for rel in relationships:
            content = format_relationship_content(
                rel.get("source", ""),
                rel.get("target", ""),
                rel.get("description", "")
            )
            if content:
                context.append({
                    "type": "relationship",
                    "content": content,
                    "src_id": rel.get("source", ""),
                    "tgt_id": rel.get("target", ""),
                    "description": rel.get("description", "")
                })

        return BenchmarkQueryResult(
            answer=ensure_string(answer, default="无法生成答案"),
            context=context
        )
```

**关键原则**：
1. **不抛出异常**：类型转换失败时使用默认值，而不是抛出错误
2. **灵活处理**：接受多种输入类型，统一转换为期望格式
3. **参考实际实现**：与评估脚本的处理方式保持一致

### 3. 类型转换工具

```python
def ensure_string(value: Any) -> str:
    """确保返回字符串"""
    if value is None:
        return ""
    if isinstance(value, str):
        return value
    return str(value)

def ensure_string_list(value: Any) -> List[str]:
    """确保返回字符串列表"""
    if value is None:
        return []
    if isinstance(value, str):
        return [value] if value else []
    if isinstance(value, list):
        return [ensure_string(item) for item in value]
    return [str(value)]

def ensure_dict(value: Any) -> Dict[str, Any]:
    """确保返回字典"""
    if value is None:
        return {}
    if isinstance(value, dict):
        return value
    return {"value": value}
```

## 常见类型错误和解决方案

### 1. context字段类型错误

**错误示例**：

# ❌ 错误：context是字符串列表（旧格式）

    "answer": "答案",
    "context": ["片段1", "片段2"]  # 缺少统一格式！

## 总结

### 必须的类型要求

#### 传入参数
1. **字符串参数**：question、content、working_dir、model_name等
2. **整数参数**：top_k、chunk_size、chunk_overlap等
3. **可选参数**：使用Optional[T]类型提示

#### 返回数据
1. **answer字段**：必须是字符串（str）
2. **context字段**：必须是统一格式的字典列表（List[Dict[str, Any]]），每个字典包含 `type` 和 `content` 字段
3. **所有字段**：必须存在，类型正确

### 核心原则

1. **类型一致性**：传入参数和返回数据的类型必须一致
2. **统一格式**：context字段使用统一格式，包含 `type` 和 `content` 字段
3. **语义保留**：实体和关系保留语义字段（entity_name, entity_type, src_id, tgt_id 等）
4. **格式化工具**：使用 `format_entity_content()` 和 `format_relationship_content()` 格式化内容
5. **错误安全**：类型错误时返回安全的默认值，不抛出异常

### 与评估脚本的兼容性

**关键点**：
- 评估脚本通过 `BenchmarkQueryResult.get_context_list()` 方法提取纯文本列表
- 统一格式保留了语义信息，同时支持评估脚本的文本提取需求
- 实体格式：`"实体名 (类型): 描述"`
- 关系格式：`"起点->终点: 描述"`
