## 🚀 使用步骤

### 步骤 1: 安装依赖

```bash
pip install datasets pandas pyarrow
```

### 步骤 2: 下载和转换数据集

```bash
python hotpotqa/download_and_convert.py
```

**预期输出**:
- 下载 HotpotQA validation 集（约 7405 条数据）
- 转换为评估框架格式
- 生成 50 个文档，200 个问题

### 步骤 3: 运行评估

#### 使用 ClearRAG

```bash
# 完整流程（构建 + 查询）
python Examples/run_clearrag.py \
  --subset hotpotqa \
  --config_path ClearRAG/config/config.yaml

# 跳过构建（如果已构建过）
python Examples/run_clearrag.py \
  --subset hotpotqa \
  --config_path ClearRAG/config/config.yaml \
  --skip-build
```

#### 使用 LightRAG

```bash
python Examples/run_lightrag.py \
  --subset hotpotqa \
  --mode API \
  --model_name deepseek-chat \
  --llm_base_url https://api.deepseek.com/v1 \
  --embed_provider zhipu \
  --embed_model embedding-3 \
  --retrieve_topk 5 \
  --base_dir ./lightrag_workspace
```

## 📊 数据集特点

### 规模对比

| 特性 | HotpotQA | Medical |
|------|----------|---------|
| 文档数 | 50 | 1 (但很大) |
| 问题数 | 200 | 2062 |
| 问题类型 | Complex Reasoning | 4 种类型 |
| 推理需求 | 多跳推理 | 单跳+多跳 |

### 数据格式

**语料库** (`Datasets/Corpus/hotpotqa.parquet`):
```json
{
  "corpus_name": "HotpotQA",
  "context": "完整的维基百科文章文本..."
}
```

**问题集** (`Datasets/Questions/hotpotqa_questions.json`):
```json
{
  "id": "HotpotQA-xxx",
  "source": "HotpotQA",
  "question": "问题文本",
  "answer": "答案文本",
  "question_type": "Complex Reasoning",
  "evidence": "支持事实的句子..."
}
```

## ⚠️ 注意事项

1. **首次下载**: 数据集较大，首次下载需要时间
2. **内存使用**: 处理完整数据集需要足够内存
3. **问题类型**: 所有问题标记为 "Complex Reasoning"（多跳推理）
4. **文档去重**: 多个问题可能共享相同的文档

## 🔧 自定义规模

编辑 `hotpotqa/download_and_convert.py`:

```python
# 修改这些常量
TARGET_CORPUS_SIZE = 50      # 语料库文档数
TARGET_QUESTIONS_SIZE = 200  # 问题数
```

## 📈 验证数据

转换完成后验证：

```python
import pandas as pd
import json

# 检查语料库
corpus = pd.read_parquet('Datasets/Corpus/hotpotqa.parquet')
print(f"语料库文档数: {len(corpus)}")
print(f"平均文档长度: {corpus['context'].str.len().mean():.0f} 字符")

# 检查问题
with open('Datasets/Questions/hotpotqa_questions.json', 'r', encoding='utf-8') as f:
    questions = json.load(f)
print(f"问题数: {len(questions)}")
print(f"示例问题: {questions[0]}")
```

## 🐛 故障排除

### 问题 1: 下载失败

**解决方案**:
```bash
# 设置 HuggingFace 缓存目录
export HF_HOME=./hf_cache
python hotpotqa/download_and_convert.py
```

### 问题 2: 内存不足

**解决方案**: 减少目标规模
```python
TARGET_CORPUS_SIZE = 30
TARGET_QUESTIONS_SIZE = 100
```

### 问题 3: 类型错误

如果遇到 `TypeError: string indices must be integers`，脚本已添加类型检查和错误处理，会自动跳过异常数据。
