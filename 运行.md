## 参数说明

### 共同参数
- `--subset`: 选择数据集 (medical, novel, medical_test, hotpotqa)
- `--sample N`: 仅处理N个样本（用于快速测试）
- `--config_path`: ClearRAG配置文件路径

### LightRAG 特有参数
- `--mode`: 运行模式 (API)
- `--model_name`: LLM模型名称
- `--llm_base_url`: LLM API地址
- `--embed_provider`: 嵌入模型提供商
- `--embed_model`: 嵌入模型名称
- `--retrieve_topk`: 检索返回的文档数量
- `--base_dir`: 工作目录
- `--skip-build`: 跳过索引阶段

### ClearRAG 特有参数
- `--top_k`: 查询时返回的上下文数量（默认值）
- `--top_k_sentences`: 返回的句子数量（可选，如果未指定则使用--top_k值）
- `--top_k_passages`: 返回的段落数量（可选，如果未指定则使用--top_k值）
- `--enable_passage_retrieval`: 是否启用段落检索（默认启用）
- `--disable_passage_retrieval`: 禁用段落检索（仅返回句子和图检索结果）
- `--resume-from-chunk N`: 从第N个切块继续处理（断点续传）

---

### ClearRAG 检索控制说明

ClearRAG 框架现在支持更精细的检索控制，允许用户独立控制句子和段落的返回数量：

1. **句子检索**：通过语义传播激活的高相关性句子，来源于原始文本
2. **段落检索**：通过Linear图检索器计算的高相关性段落，经过实体权重和PPR计算
3. **图检索**：通过Neo4j图数据库检索的结构化知识

#### 使用场景
- **学术研究**：需要更多精确句子时，设置 `--top_k_sentences 20 --top_k_passages 5`
- **快速问答**：需要较少上下文时，设置 `--top_k_sentences 5 --top_k_passages 2`
- **仅图检索**：只需要结构化知识时，使用 `--disable_passage_retrieval`
- **详细分析**：需要全面上下文时，设置 `--top_k_sentences 15 --top_k_passages 10`

## 常见问题

### 1. 缺少依赖
如果遇到 "ModuleNotFoundError" 错误，请安装相应依赖：
```powershell
pip install -r requirements.txt
# 或对于 ClearRAG 特定依赖
ClearRAG\.venv\Scripts\python.exe -m pip install datasets tqdm
```

### 2. Neo4j连接问题
确保Neo4j服务正在运行，并且ClearRAG/config/config.yaml中的连接信息正确。

### 3. 内存不足
如果遇到内存问题，可以：
- 减少并发数（在config.yaml中调整chunk_parallel部分）
- 使用更小的sample值
- 使用--skip-build参数跳过构建阶段

### 4. API限速问题
如果遇到DeepSeek API限速错误，可以：
- 降低并发数（在config.yaml中调整max_concurrency_entities和max_concurrency_relationships）
- 减少batch_size值（建议值：20-50）
- 增加重试间隔时间（在config.yaml中调整retry部分的initial_delay和max_delay）
- 切换到付费API计划以获得更高的速率限制
