#### Novel 数据集常见问题

**问题现象**：查询结果全部返回 `[no-context]`，`context` 字段为空数组 `[]`

**原因**：
- Novel数据集有20个独立语料，默认会**并发启动20个索引任务**
- 瞬间大量API请求触发限流/超时
- 索引过程被中断，文档状态停留在 `pending`，图数据未生成

**解决方案**：
1. 使用 `--sample 1` 限制处理的语料数量
2. 检查 `lightrag_workspace/Novel-XXXXX/` 目录是否有完整的图数据文件
3. 如果缺少 `graph_chunk_entity_relation.graphml`，需要重新运行索引（不使用 `--skip-build`）

## 参数说明

### 共同参数
- `--subset`: 选择数据集 (medical, novel, medical_100, hotpotqa)
- `--sample N`: 仅处理N个样本（用于快速测试）
- `--config_path`: ClearRAG配置文件路径

### LightRAG 特有参数
- `--mode`: 运行模式 (API)
- `--model_name`: LLM模型名称
- `--llm_base_url`: LLM API地址
- `--embed_provider`: 嵌入模型提供商
- `--embed_model`: 嵌入模型名称
- `--retrieve_topk`: 检索返回的文档数量
- `--base_dir`: 工作目录
- `--skip-build`: 跳过索引阶段

### ClearRAG 特有参数
- `--top_k`: 查询时返回的上下文数量（默认值）
- `--top_k_sentences`: 返回的句子数量（可选，如果未指定则使用--top_k值）
- `--top_k_passages`: 返回的段落数量（可选，如果未指定则使用--top_k值）
- `--enable_passage_retrieval`: 是否启用段落检索（默认启用）
- `--disable_passage_retrieval`: 禁用段落检索（仅返回句子和图检索结果）

### HippoRAG 特有参数
- `--mode`: 运行模式 (API 或 ollama)
- `--model_name`: LLM模型名称
- `--embed_model_path`: 嵌入模型路径或名称
- `--base_dir`: HippoRAG工作目录
- `--llm_base_url`: LLM API地址或Ollama地址
- `--llm_api_key`: LLM API密钥（可选，也可使用OPENAI_API_KEY环境变量）

### Fast-GraphRAG 特有参数
- `--subset`: 数据集选择 (`medical` 或 `novel`)
- `--mode`: LLM运行模式 (`API` 或 `ollama`)
- `--model_name`: LLM模型名称
- `--llm_base_url`: LLM API地址
- `--llm_api_key`: LLM API密钥（也可用环境变量 `LLM_API_KEY`）
- `--embed_provider`: 嵌入服务类型 (`api` 或 `local`)
  - `api`: 使用 OpenAI 兼容 API（智谱、DeepSeek 等）
  - `local`: 使用本地 HuggingFace 模型
- `--embed_model`: 嵌入模型名称（API模式）或路径（local模式）
- `--embed_base_url`: 嵌入 API 地址（默认：智谱）
- `--embed_api_key`: 嵌入 API 密钥（默认：使用 `ZHIPUAI_API_KEY` 环境变量）
- `--base_dir`: Fast-GraphRAG工作目录
- `--sample`: 限制处理的样本数量（用于测试）

### ClearRAG 检索控制说明

ClearRAG 框架现在支持更精细的检索控制，允许用户独立控制句子和段落的返回数量：

1. **句子检索**：通过语义传播激活的高相关性句子，来源于原始文本
2. **段落检索**：通过Linear图检索器计算的高相关性段落，经过实体权重和PPR计算
3. **图检索**：通过Neo4j图数据库检索的结构化知识

#### 使用场景
- **学术研究**：需要更多精确句子时，设置 `--top_k_sentences 20 --top_k_passages 5`
- **快速问答**：需要较少上下文时，设置 `--top_k_sentences 5 --top_k_passages 2`
- **仅图检索**：只需要结构化知识时，使用 `--disable_passage_retrieval`
- **详细分析**：需要全面上下文时，设置 `--top_k_sentences 15 --top_k_passages 10`

## 评估

### 评估说明

> **重要**：评估时使用的嵌入模型不需要与检索时一致。检索评估只使用LLM判断上下文相关性，生成评估使用嵌入模型计算答案语义相似度。所有框架的评估使用相同的嵌入模型（BGE）以确保公平性。

```powershell
  --data_file .\results\clearrag\predictions_Medical.json `

  --num_samples 10
```

## 常见问题

### 1. 缺少依赖
如果遇到 "ModuleNotFoundError" 错误，请安装相应依赖：
```powershell
pip install -r requirements.txt
# 或对于 ClearRAG 特定依赖
ClearRAG\.venv\Scripts\python.exe -m pip install datasets tqdm
```

### 2. Neo4j连接问题
确保Neo4j服务正在运行，并且ClearRAG/config/config.yaml中的连接信息正确。

### 3. 内存不足
如果遇到内存问题，可以：
- 减少并发数（在config.yaml中调整chunk_parallel部分）
- 使用更小的sample值
- 使用--skip-build参数跳过构建阶段

### 4. API限速问题
如果遇到DeepSeek API限速错误，可以：
- 降低并发数（在config.yaml中调整max_concurrency_entities和max_concurrency_relationships）
- 减少batch_size值（建议值：20-50）
- 增加重试间隔时间（在config.yaml中调整retry部分的initial_delay和max_delay）
- 切换到付费API计划以获得更高的速率限制

### 5. Fast-GraphRAG Embedding 加载失败
如果遇到 `HuggingFaceEmbeddingService` 未找到错误：
- 确保已按照 `Examples/README.md` 创建 `_hf.py` 文件
- 检查 `fast_graphrag/_llm/__init__.py` 是否正确导出
- 确认嵌入模型路径正确（使用 HuggingFace 模型名称如 `BAAI/bge-large-en-v1.5`）

### 6. Fast-GraphRAG/HippoRAG subset 不支持
Fast-GraphRAG 和 HippoRAG 当前仅支持 `medical` 和 `novel` 数据集，如需支持其他数据集：
- 编辑对应脚本中的 `SUBSET_PATHS` 配置
- 添加对应的数据集路径映射

### 7. 为什么评估使用不同的嵌入模型？
评估时使用的嵌入模型（BGE）与检索时使用的嵌入模型（如智谱embedding-3）可以不同，原因如下：
- **检索评估**：只使用LLM判断上下文相关性，不使用嵌入模型
- **生成评估**：使用嵌入模型计算答案与标准答案的语义相似度，这是独立的评估任务
- **公平性保证**：所有框架的答案都用同一个嵌入模型评估，确保公平性
